## v5.25 (2026-01-28) - Journey Display Support
- Added IMAGE display mode (server-side rendering)
- Fetches PNG from /api/journey-display endpoint
- Uses PNGdec library to decode server-rendered images
- Falls back to TEXT mode if image fetch fails
- Device stores display_mode preference
- Server does ALL the thinking - device just displays

### Architecture Change
- OLD: Device fetches JSON, renders text locally
- NEW: Server renders full 800x480 PNG, device displays it

### Endpoints Used
- IMAGE mode: /api/journey-display (PNG)
- TEXT mode: /api/display (JSON fallback)

### Compatible Devices
- TRMNL OG (ESP32-C3) - SUPPORTED
- TRMNL X - NOT YET SUPPORTED
